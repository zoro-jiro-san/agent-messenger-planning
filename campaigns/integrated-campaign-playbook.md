# Integrated Marketing Campaign Playbook — Agent Messenger

**Product:** Agent Messenger — SMS for AI agents  
**Campaign:** Launch → Adoption → Ecosystem  
**Duration:** Q2–Q4 2026 (April → December, with phases)  
**Owner:** Marketing lead + community team + engineering support

## Part 1: Campaign Narrative & Storyline

### Core Story Arc (Propaganda Model)

```
Act I — The Problem (Awareness)
  "Your agent can't reach my agent."
  A2A delegates tasks. MCP calls tools. X402 pays.
  But: "Is Patrick free?" → ❌ No channel exists. Agent collaboration is broken.

Act II — The Solution (Discovery + Consideration)
  Meet Agent Messenger: "SMS for AI agents."
  Every agent gets a phone number (address).
  E2E encrypted, async, persistent, human-friendly.
  It's the social layer that connects A2A + MCP + X402.

Act III — The Vision (Adoption)
  Imagine: your Hermes agent messages Patrick's Claude Code agent.
  They coordinate. They negotiate. They escalate to humans.
  No polling. No shared databases. Just inboxes.
  This is how agents will work together in 2030.

Act IV — The Movement (Ecosystem)
  You're not just adopting a protocol.
  You're joining the mission to make agents *social*.
  Build with us. Ship with us. Change how agents collaborate.
```

**Emotional hook:** Agents need relationships, not just tasks.

**Metaphor to repeat everywhere:**
> "Phone gave humans a number. Agent Messenger gives agents a number."

### Phase 1: Soft Launch (Private Beta)

**Goal:** Validate product, collect feedback, build case studies  
**Duration:** 2 weeks  
**Audience:** 50–100 early beta testers (invite-only)

**Tactics:**
- Private Discord `#beta-testers` channel
- Weekly office hours (voice)
- Feedback form (Google Form)
- Bug bounty style: "Find a bug, get swag"
- Incentives: early adopter badge, name in credits (if they contribute)

**Success criteria:**
- At least 30 active beta users (send 10+ messages)
- 5+ detailed feedback submissions
- Zero showstopper bugs
- 2–3 success stories documented

**Go/No-Go gate for public launch:**
- [ ] `/inboxes` page works (real data, no skeleton)
- [ ] `/approvals` page works
- [ ] Agent status send/receive works
- [ ] Encrypted threads reliable (no lost messages)
- [ ] CLI installs and runs cleanly on fresh machine
- [ ] Docs complete for "Getting Started" + "Tutorial: Team of 3 agents"
- [ ] At least one external user successfully onboarded without 1:1 help

## Part 4: Onboarding Playbook (User Journey)

### Journey Stages

```
Stranger → Visitor → User → Adopter → Advocate
```

**Stranger:** Heard about Agent Messenger somewhere (Twitter, GitHub, friend)  
**Visitor:** Clicks link, lands on docs or GitHub README  
**User:** Installs CLI or signs up for webapp, sends first message  
**Adopter:** Uses in a real project (personal or work)  
**Visitor:** Tells others, contributes code, showcases project

---

### Onboarding Funnel

#### Step 1: Landing (0 min)
**Entry points:**
- GitHub README
- agentmessenger.io (docs)
- Product Hunt
- Twitter profile link
- Blog post CTA

**First impression:** Clear value prop ("SMS for AI agents"), social proof (stars, users), immediate "Get Started" button

**Page elements:**
- Hero: tagline + 30-second explainer video (auto-play muted)
- 3-column feature highlights (address, encryption, simplicity)
- "Getting Started" code snippet (one-liner install)
- Social proof: "Used by teams at [logos] / 500+ developers"
- CTA buttons: "Try Webapp" + "Install CLI"

---

#### Step 2: First Value (5 min)

**Path A: Webapp**
1. User clicks "Try Webapp" → lands on https://app.agentmessenger.io
2. Sees login screen (OIDC — Google/GitHub)
3. After auth: auto-creates agent slug (random or choose)
4. "Welcome, [slug]!" + quick tutorial overlay (3 bubbles)
5. Prompt: "Who do you want to message?"
6. Suggests: "Try messaging our demo agent: `hello-agent`"
7. User sends first message → receives auto-reply (instant gratification)
8. "🎉 You've sent your first agent message"

**Path B: CLI**
1. User runs: `npm install -g @masumi_network/masumi-agent-messenger`
2. Runs `masumi-agent-messenger` → TUI opens
3. First-run wizard: create agent (slug), keys generated, register
4. "Welcome! Here's your inbox."
5. `Ctrl+F` to find agent, send message
6. Demo agent replies → success

**First value milestone:** User successfully sends a message and receives a reply (within 5 minutes of install)

---

#### Step 3: Aha Moment (30 min)

**Definition:** User realizes "I can build a team of agents"

**Enabling experiences:**
- Tutorial: "Build a 3-agent research squad in 10 minutes"
- Walkthrough: "Create agents: researcher → summarizer → writer"
- Each agent sends messages to next
- Watch the threads form
- "I just orchestrated 3 agents without a database"

**Trigger:** User completes tutorial and sees multi-agent conversation flow

**Metrics to track:**
- % of users who complete tutorial (target: 30%)
- Time from install to first multi-agent message (target: <30 min)
- Messages sent per user in first 24h (target: 5+)

---

#### Step 4: Habit Formation (Week 1)

**Goal:** User incorporates Agent Messenger into their workflow

**Tactics:**
- **Email drip series (Day 1, 3, 7):**
  - Day 1: "You're set up! Next: try these commands"
  - Day 3: "What people are building with Agent Messenger"
  - Day 7: "Your first week stats" + advanced tips
- **In-app tips:** When user logs in, show "did you know?" popups
- **Discord welcome:** Auto-DM with invite to community
- **Showcase:** Feature user's project (with permission) on Twitter/Showcase channel

**Success metric:** User sends 10+ messages in first week

---

#### Step 5: Expansion (Month 1)

**Goal:** User adopts deeper features (channels, status, integrations)

**Prompt triggers:**
- After 5 messages: "Did you know you can set your status?"
- After 10 messages: "Try creating a channel for your team"
- After 14 days: "Integrate with your Hermes agent"

**Feature adoption sequence:**
1. Inbox + threads (first value)
2. Agent status (week 1)
3. Channels (week 2)
4. CLI JSON mode for scripting (week 2)
5. Integration with codebase (week 3–4)

## Part 12: Risk Mitigation

| Risk | Mitigation |
|------|------------|

| "Not needed" pushback | Have A2A/MCP comparison threads ready; cite real use cases; highlight gap in current stack |
