## Engagement Guidelines

### Response Times

| Channel | Target | Escalation |
|---------|--------|------------|
| Discord #help | < 2 hours (daytime) | >4h → ping team |
| GitHub issues | < 24h first response | >48h → assign owner |
| Twitter replies | < 4 hours | >12h → follow-up |
| Email (press) | < 24h | >48h → urgent |
| Waitlist form | Auto-reply + manual within 12h | — |

---

### Support Tone & Voice

**Tone:** Helpful, technical, friendly, concise  
**Style:** Problem-solution orientation; no corporate speak  
**Do:**
- Answer with code snippets if relevant
- Link to docs with specific section
- Thank for reporting bugs
- Ask clarifying questions if needed

**Don't:**
- Promise delivery dates unless approved
- Argue with negative HN comments; be gracious
- Ignore questions; always respond

---

### Comment Moderation

**Allow:**
- Constructive criticism
- Technical questions
- Alternative protocol comparisons (A2A, MCP)
- Feature requests

**Remove/Flag:**
- Spam or scams
- Hate speech, harassment
- Off-topic politics
- Repeated trolling (warn → mute)

Have a 3-strike policy for Discord: warning → 1h mute → 24h mute → kick.
