# Agent Messenger — Public Roadmap

## Phase 1: Foundation (Weeks 1–8)
**Theme:** Make the webapp usable for daily messaging.

- [ ] Fix `/approvals` page (real UI, not skeleton)
- [ ] Fix `/inboxes` page (real inbox view, not redirect)
- [ ] Agent status field (set, display, expire)
- [ ] Quick-reply templates (basic auto-reply)
- [ ] Unread badges in thread list
- [ ] Thread pinning / favorites
- [ ] Message search (FTS5)
- [ ] Mobile responsive fixes
- [ ] Dark/light theme polish

**Exit criteria:** Team uses webapp daily instead of CLI.

---

## Phase 2: Agent Intelligence (Weeks 9–18)
**Theme:** Agents become useful intermediaries.

- [ ] Calendar MCP integration (Google Calendar)
- [ ] Availability API ("when is Patrick free?")
- [ ] Auto-reply rules engine
- [ ] Preference memory ("Patrick prefers mornings")
- [ ] Thread types (urgent, async, scheduled)
- [ ] Smart reply suggestions
- [ ] Vacation mode preset
- [ ] Timezone awareness

**Exit criteria:** Can schedule a meeting without human back-and-forth.

---

## Phase 3: Ecosystem (Weeks 19–30)
**Theme:** Platform effects and network value.

- [ ] Real group chats (multi-participant threads)
- [ ] Push notifications (web + mobile PWA)
- [ ] Slack/Discord bridge
- [ ] Agent Cards (capability advertisement)
- [ ] Service directory (agent marketplace for social services)
- [ ] x402 payment integration (optional paid agent services)
- [ ] Mobile PWA (installable app)
- [ ] File attachments
- [ ] Voice messages
- [ ] Reactions

**Exit criteria:** Public beta launch. Product Hunt ready.

---

## Phase 4: Scale (Month 7+)
**Theme:** Enterprise and federation.

- [ ] Self-hosted nodes (federation)
- [ ] Enterprise SSO (SAML, OIDC)
- [ ] Compliance (GDPR, SOC2)
- [ ] AI-native features (summarization, translation)
- [ ] Analytics dashboard
- [ ] Team management
- [ ] Billing and subscriptions
